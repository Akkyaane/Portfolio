TARS 2

Contexte

Dans le cadre du deuxième stage de BTS SIO, qui s'est déroulé du 08 janvier au 09 février 2024, j'ai eu l'opportunité de travailler sur le développement d'un assistant virtuel nommé TARS 2 au sein de l'entreprise Kozea group. Spécialisé dans le milieu médical, Kozea group accompagne et facilite le travail des professionnels de santé en proposant des services digitaux. De la création et de la promotion de sites Internet incluant différentes fonctionnalités (envoi d'ordonnances, vente de produits, Click and Collect, prise de rendez-vous) pour les établissements de santé à la gestion du tiers-payant ou des réseaux sociaux : tout est pensé pour permettre aux professionnels de santé de se recentrer sur leur cœur de métier.

Afin d'optimiser l'efficacité de la plateforme interne de l'entreprise, l'assistant virtuel TARS 2 a pour objectif de rendre plus intuitif la recherche et la manipulation des données en étant capable d'altérer ses données et de répondre aux requêtes utilisateurs.

Technologies

  • Environnement : Linux
  • Interface d'administration de bases de données : DBeaver
  • Interface de programmation d'application : OpenAI API
  • Frameworks : Flask, SQLAlchemy
  • Langages : CSS, HTML, Python
  • Librairie : Tailwind CSS
  • Système de gestion de bases de données : MySQL
  • Versioning : Git, GitHub
  • Éditeur de texte : Visual Studio

Côté back-end, l'application a été développée en langage Python et à l'aide du framework Flask (basé sur Python), ainsi que de l'interface de programmation d'application OpenAI API. L'application a nécessité l'utilisation et la gestion d'une copie de la base de données de l'entreprise avec DBeaver et MySQL, et la gestion des requêtes avec SQLAlchemy, un framework également basé sur Python.

Comme TARS 1, une interface de type chatbot, développée avec les langages HTML et CSS, ainsi que la librairie Tailwind CSS, a été intégrée au projet pour mettre en avant le côté conversationnel entre l'utilisateur et l'intelligence artificielle.

Fonctionnement

Pour faire fonctionner TARS 2, il est nécessaire de créer des fonctions en amont. Ces fonctions réalisent des actions précises et requièrent, pour certaines, l'utilisation d'une base de données afin de manipuler certaines données. L'assistant est conçu pour appeler ces fonctions quand il en a besoin. Considéré comme des outils, les fonctions sont décrites une par une dans un fichier : leurs objectifs, leurs paramètres, etc. Le but est d'expliciter ces outils pour permettre à l'intelligence artificielle de comprendre quelles fonctions utiliser pour répondre aux requêtes de l'utilisateur.

Une fois les outils en main, il faut créer un assistant selon la documentation d'OpenAI API. Un thread, autrement dit une liste dans laquelle va se trouver l'ensemble de la conversation entre l'utilisateur et l'intelligence artificielle, est créé. À chaque message envoyé par l'utilisateur, l'assistant détermine s'il doit appeler des fonctions ou non pour répondre. Selon le contexte, il détermine aussi la quantité de fonctions à appeler.

Cette base permet ainsi à l'assistant virtuel d'être opérationnel dans un environnement de travail qui est tel qu'il peut interroger, insérer, modifier ou supprimer des données d'une base de données.

Conclusion

Cette seconde expérience, axée aussi bien sur le front que sur le back, m'a permis d'en apprendre davantage sur le métier de développeur full-stack et de confirmer mon intérêt pour ce métier.
