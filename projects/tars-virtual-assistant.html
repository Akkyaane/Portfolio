TARS 2

Contexte

Dans le cadre du second stage de BTS SIO, qui s'est déroulé du 08 janvier au 09 février 2024, j'ai eu l'opportunité de travailler sur le développement d'un assistant virtuel nommé TARS 2 au sein de l'entreprise Kozea group. Spécialisé dans le milieu médical, Kozea group accompagne et facilite le travail des professionnels de santé en proposant des services digitaux. De la création et de la promotion de sites Internet incluant différentes fonctionnalités (envoi d'ordonnances, vente de produits, Click and Collect, prise de rendez-vous) pour les établissements de santé à la gestion du tiers-payant ou des réseaux sociaux : tout est pensé pour permettre aux professionnels de santé de se recentrer sur leur cœur de métier.

Afin d'optimiser l'efficacité de la plateforme interne de l'entreprise, l'assistant virtuel TARS 2 a pour objectif de rendre plus intuitif la recherche et la manipulation des données.

Technologies

Plusieurs outils ont été utilisés :

  • Langages, frameworks et librairies : Python, HTML, CSS, SQLAlchemy, Flask, Tailwind CSS
  • Éditeur de texte : Visual Studio
  • Interface d'administration et système de gestion de base de données : DBeaver, MySQL
  • Versioning : Git, GitHub
  • Environnement : Linux

Côté back-end, l'application a été développée en Python, et à l'aide de l'API d'OpenAI et SQLAlchemy, ce dernier permettant de manipuler les données d'une base de données.

Côté front-end, l'interface a été développée avec Flask, un framework basé sur le langage Python. Comme TARS 1, une interface de type chatbot, développée avec la librairie Tailwind CSS, a été intégrée au projet pour mettre en avant le côté conversationnel entre l'utilisateur et l'intelligence artificielle.

Fonctionnement

Pour faire fonctionner TARS 2, il est nécessaire de créer des fonctions en amont. Ces fonctions réalisent des actions précises et requièrent, pour certaines, l'utilisation d'une base de données pour manipuler certaines données. L'assistant, fonctionnant sur la base de ChatGPT 4, est conçu pour appeler ces fonctions quand il en a besoin. Considéré comme ses « outils », les fonctions sont décrites une par une dans un fichier : leur objectif, leurs paramètres, etc. L'objectif est d'expliciter ses outils pour permettre à l'intelligence artificielle de comprendre de quoi elle a besoin pour répondre aux requêtes de l'utilisateur. Une fois les outils en main, il faut créer un assistant selon la documentation de l'API d'OpenAI. Puis, un thread, autrement dit une liste dans laquelle va se trouver l'ensemble de la conversation entre l'utilisateur et l'intelligence artificielle, est créé. À chaque message envoyé par l'utilisateur, l'assistant détermine s'il doit appeler des fonctions ou non pour répondre. Selon le contexte, il détermine aussi la quantité de fonctions à appeler. Cette base permet ainsi à l'assistant virtuel d'être opérationnel dans un environnement de travail qui est tel qu'il peut interroger, insérer, modifier ou supprimer des données d'une base de données.

Conclusion

Cette seconde expérience, axée aussi bien sur le front que sur le back, m'a permis d'en apprendre davantage sur le métier de développeur full-stack et de confirmer mon intérêt porté pour le métier de développeur.
